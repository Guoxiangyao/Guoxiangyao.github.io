Mysql之数据类型以及属性

发表于 2017-02-04 | 分类于 Mysql | | 阅读次数

这篇博客主要是讲解Mysql的数据类型以及属性。

一、Mysql的数据类型
1.整型(xxxint)
(1)tinyint(m)：一个字节表示（-128~127）
(2)smallint(m)：两个字节表示（-32768~32767）
(3)mediumint(m)：三个字节表示（-8388608~8388607）
(4)int(m)：四个字节表示（-2147483648~2147483647）
(5)bigint(m)：八个字节表示 （正负9.22*10的18次方）
注：m表示最大显示的宽度，起到格式控制的作用，与实际的值没有关系

2.浮点型(float和double)
(1)float(m,d) 单精度浮点型，8位精度（4字节），m是十进制数字的总个数，d是小数点后面的数字个数
(2)double(m,d) 双精度浮点型，16位精度（8字节）参数m只影响显示效果，不影响精度d却不同，会影响到精度，涉及到小数的舍去问题

3.定点数(decimal)
decimal(m,d):浮点型在数据库中存放的时近似值，而定点类型在数据库中存放的是精确值。参数m是定点类型数字的最大个数，范围为0~65，d是小数点右侧数字的个数，范围为0~30，但不得超过m。

4.字符串
(1)char(n) ：固定长度的字符串，最多255个字符
(2)varchar(n)： 固定长度的字符串，最多65535个字符
(3)tinytext：可变长度字符串，最多255个字符
(4)text：可变长度的字符串，最多65535个字符
(5)mediumtext：可变长度的字符串，最多2的24次方-1个字符
(6)longtext：可变长度字符串，最多2的32次方-1个字符
注：char类型如果实际位数小于n，则会在前面补齐空格,varchar类型如果实际位数小于n，则不会存在补齐的情况.

5.二进制数据(xxxBlob)
xxxBlob和xxxtext是对应的，不过存储方式不同，xxxtext是以文本方式存储的，如果存储英文的话区分大小写，而xxxblob是以二进制方式存储的，不区分大小写。xxxblob存储的数据只能整体读出，xxxtext可以指定字符集，xxxblob不用指定字符集。

6.日期时间类型
(1)date：日期‘2008-12-1’
(2)time：时间‘12:25:36’
(3)datetime：日期时间2008-12-1 12:25:36
(4)timestamp ：不固定
timestamp比较特殊，如果定义一个字段类型为timestamp，这个字段的时间会在其他字段修改的时候自动刷新。所以这个数据类型可以存放这条记录最后被修改的时间，而不是真正来存放时间的。
注：在数据库表示字符串用单引号

二、数据类型的属性
(1)NULL：数据列可以包含NULL值
(2)NOT NULL：数据列不允许包含NULL值
(3)DEFAULT xxx：默认值，如果插入记录的时候没有指定值，将取这个默认值
(4)PRIMARY KEY：主键 唯一代表记录的
(5)AUTO_INCREMENT：递增，如果插入记录的时候没有指定值，则在上一条记录的值上加1，仅适用于整数类型
(6)UNSIGNED：无符号
(7)CHARACTER SET name：指定一个字符集
